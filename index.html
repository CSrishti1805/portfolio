How it works

From question to insight
in seconds.

💬

Step 01

User asks a question

A product manager types a question about toddler bike trends. The system classifies it in one LLM call before spending any budget.

Query: "What's trending in toddler balance bikes?"
Triage: RESEARCH → proceed to tool layer
🧠

Step 02

Agent selects tools

The LLM router selects the best available provider. The orchestrator plans which tools to call — all in a single batch turn.

Provider: Gemini 2.5 Flash (200 OK)
Planned: check_db + amazon + youtube + news + history
⚡

Step 03

Concurrent execution

All tools run simultaneously via asyncio.gather. Amazon, YouTube, and News APIs are fetched in parallel — not sequentially.

Amazon scrape: 5 listings · 6.5s
YouTube signals: 5 videos · 4.0s
Total wall time: 6.5s (parallel, not 14.5s)
🔍

Step 04

Context pipeline

Results pass through relevance filtering, deduplication, and token budget control before reaching the LLM — keeping quality high and cost low.

Raw results: 12 items → filtered to 7 relevant
Token budget: 3,400 input → within limit
📋

Step 05

Structured briefing

The agent synthesises a 8-section structured report with evidence tags, confidence levels, and source attribution — persisted to PostgreSQL for future comparison.

Output: Key Trends · Strategic Insights · Opportunities
Saved to DB · Compared against prior run · Done in 28s
